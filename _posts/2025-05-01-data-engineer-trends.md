### 데이터 엔지니어링 토론 주제

#### 1. 데이터 엔지니어링 흐름 읽기

* 목표 : 최신 기술에 대한 이해와 도입을 검토할 수 있는 기회를 제공
* 대상 : 엘지전자 데이터 중급 엔지니어

* 목차 : 
  * 데이터 엔지니어링의 역할이 단순한 ETL을 넘어서고 있음
    * 예전처럼 단순히 데이터를 **수집 → 변환 → 적재(ETL)**하는 것을 넘어서서, 데이터 엔지니어가 기업 내 **데이터 플랫폼의 핵심 아키텍트**이자 **운영 자동화의 중심**, 나아가 **데이터 기반 의사결정의 기반을 구축하는 역할**까지 확장되었다
  * 데이터 엔지니어링 기술 트렌드 소개
    * 이제는 Lambda Architecture보다 Medallion Architecture가 대세
    * **Raw → Cleansed → Aggregated** 계층 구분된 데이터 저장소 설계 가능
    * Delta Lake, Apache Iceberg 같은 최신 테이블 포맷 도입 시 판단력 확보
    * 배치와 스트리밍 통합 처리를 위해 Spark Structured Streaming을 활용할 수 있는 능력 확보
  * 기술 트렌드의 변화가 실무에 미치는 영향
    * 
* 조직별 상황에 맞는 기술 로드맵 검토
* 질문과 답변

### 멀티 캠퍼스 데이터 엔지니어링 과정 보완

#### 1. 도입부 강화: 학습 목표와 수강자 기대 정렬

* 강의 초반에 **"이 강의를 수강하면 어떤 문제를 해결할 수 있게 되는가?"**에 대한 구체적 안내 필요
  * 현업에서 데이터 파이프라인을 설계하고 운영할 수 있다
  * Kafka 기반의 실시간 스트리밍 처리 구축이 가능하다

#### 2. 데이터 모델링과 SQL 최적화 추가 고려

* 실무에서 중요하지만 종종 간과되는 **데이터 모델링(Basic/Star Schema)**, **SQL 튜닝/쿼리 최적화** 관련 내용이 미흡
  * Hive나 Presto 쿼리 실습 시 적절한 쿼리 최적화 전략을 병행하면 더 강력한 교육이 될 수 있음

#### 3. 클라우드 환경 관련 요소 보완

* 현재 온프레미스 또는 로컬 중심 실습 환경에 집중되어 있음
  * 클라우드 환경(GCP/AWS)의 최소 구성 및 차이점 소개 (예: S3 ↔ HDFS, Glue ↔ Airflow 등)를 비교 소개하면 현대 실무 환경과의 연결성이 향상됨

### 멀티 캠퍼스 수정 검토

#### 1.과정 소개

1.  데이터 엔지니어링 실무에 필요한 개념과 기술을 학습할 수 있는 과정입니다.
   1. 데이터 엔지니어링 실무에 꼭 **필요한 핵심 개념과 기술을 체계적으로 학습**하는 과정입니다.
2.  Sqoop, Spark, Hive 오픈소스 도구 중심으로 대용량 데이터 파이프라인을 구축 및 설계할 수 있습니다
   1. Sqoop, Spark, Hive 등 주요 오픈소스 도구를 활용해 **대용량 데이터 파이프라인을 직접 구축하고 설계** 할 수 있습니다
3.  데이터 파이프라인을 구축한 지표 분석 프로젝트가 포함되어, 신입~5년차 데이터 엔지니어가 한 단계 성장할 수 있도록 설계되었습니다
   1. 실습 기반의 지표 분석 프로젝트를 통해 **신입부터 5년차까지 데이터 엔지니어가 한 단계 더 성장할 수 있도록 구성**되어 있습니다.

#### 2. 학습 목표

1. 오픈소스 기반 데이터 엔진(Sqoop, Spark, Hive)의 내부 동작 원리를 이해할 수 있다
   * 오픈소스 기반 데이터 엔진(Sqoop, Spark, Hive)의 **핵심 기능과 내부 동작 원리를 이해하고 설명**할 수 있다
2. 데이터 수집(Ingestion), 변환(Processing), 적재(Storage) 과정을 실습 예제를 통해 직접 개발할 수 있다
   * 실습 예제를 통한 데이터 수집, 변환 및 적재 과정을 학습하고, **데이터 파이프라인을 직접 설계하고 운영**할 수 있다

2. 효율적인 데이터 파이프라인을 설계하고, 주어진 요구사항에 맞는 최적의 시스템 아키텍처를 구성할 수 있다. 
   * 주어진 요구사항에 맞는 효율적인 데이터 파이프라인 실습를 통해, **최적의 데이터 아키텍처를 설계**할 수 있다
