### 2. 몽고디비 아키텍처

> 학습 관련 영상 링크와 내용 요약

#### 2-1. 참고 사항

* 지연 복제 세트 구성원 : oplog 복제 구성 시에 약 1시간 정도 지연 복제를 구성하고 이력 관리 + 우발적인 삭제 실수에 대한 신속한 복구가 가능함

* 쓰기, 읽기 선호도 가운데 majority 와 linearizable (극단적인 최신 데이터 읽기 지원) 의 차이 이해하고 활용할 것

  * 몽고 템플릿을 사용하더라도 순서가 중요한 경우 write majority + causal consistency + read majority 조합을 사용하길 추천
  * 몽고 템플릿을 설정 파일을 통해 초기화 하여 구현하는 것은 좋으나, 요구 사항에 따라 다르게 설정하는 경우는 세션을 쓰는 것을 추천
  * 오토 클로저블 구간 내에서 세션을 생성하고 해당 스코프 내에서 적절한 읽기, 쓰기 수준을 지정하는 것이 가장 적절한 구현 방식임

* ```java
  // 요청 단위 세션 + w:"majority" + readConcern:"majority" + readPref: primary
  MongoClient client = mongoTemplate.getMongoDatabaseFactory()
                                    .getMongoDatabase()
                                    .getMongoClient();
  
  try (ClientSession session = client.startSession()) {
    return mongoTemplate.withSession(() -> session).execute(action -> {
      var coll = action.getDb().getCollection("orders")
                       .withWriteConcern(WriteConcern.MAJORITY)
                       .withReadConcern(ReadConcern.MAJORITY);
  
      // write
      coll.insertOne(session, new Document("_id", id).append("status", "created"));
  
      // read (same session → RYW 보장)
      return coll.find(session, Filters.eq("_id", id)).first();
    });
  }
  
  ```
